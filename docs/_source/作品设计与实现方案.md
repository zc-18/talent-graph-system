## 一、项目背景与痛点

在国家从"人口红利"向"人才红利"转型的关键期，"新质生产力""人工智能+"与数字经济推动以新一代信息技术为代表的新兴领域快速发展，技术迭代速度远超人才培养周期，劳动力市场出现严重"结构性矛盾"：

- **企业侧**："招不到合适的人"，人岗匹配度低、新兴岗位识别难，招聘与培养成本高；
- **人才侧**：新兴领域青年职业路径模糊、技能需求动态性强，缺乏动态行业技能图谱指引；
- **方法侧**：传统人才盘点依赖关键词匹配，对技术趋势无实时感知，难以回答"技术 A 的爆发会给岗位 B 带来哪些新技能需求"。

更深层的数据难题在于：招聘 JD（Job Description）普遍存在 **时滞（lag）、噪音（noise）、抄袭（plagiarism）、能力通胀（inflation）**；同时大模型在生成能力定义时存在 **"幻觉"** 风险，直接生成的岗位能力不可信、不可溯源。

**本系统的目标**：以"数据驱动 + 大模型 + 知识图谱"为核心，构建一个能够自我进化的"人才能力大脑"，实现从"静态画像"到"动态感知"的跨越，形成 **"多源数据采集 → 新岗位发现与定义 / 既有岗位能力更新 → 能力图谱动态构建 → 简历解析 → 精准匹配与差距分析 → 改进建议与学习路径"** 的全流程闭环。

## 三、核心功能实现

### 3.1 新岗位发现与定义

**目标**：识别市场上萌芽/兴起中、尚未标准化的新岗位，并生成结构化定义（岗位名称、核心职责、必备技能、加分技能、典型行业应用场景），支持人工优化与动态更新。

**实现流程**（`services/discovery.py`）：
1. **候选发现**：内置新兴岗位种子（提示词工程师、AI 智能体开发工程师、RAG 工程师、具身智能工程师、MLOps 工程师等），或用户自定义关键词；
2. **多源证据采集**：Tavily + Serper.dev **两个独立来源**并行检索，按 URL 去重，统计独立来源数；近 180 天新闻检索评估"新兴度"；
3. **RAG 接地生成**：将检索到的证据材料作为上下文，约束大模型"只基于证据定义、禁止臆造"，生成结构化岗位定义；
4. **能力项交叉验证**：对每个生成的技能，校验其是否在证据文本中出现，据此提升/降低置信度，标注 `web_verified`；
5. **入库**：写入 `is_new=True` 的岗位节点，全景图谱中以琥珀色高亮。

**输入输出示例**：见 `docs/测试数据/新岗位_AI智能体开发工程师_图谱与数据源.json`（输入关键词 → 15 条多源证据 → 含 7 项必备技能的完整定义）。

### 3.2 既有岗位能力动态更新（演化）

**目标**：针对现有岗位（如 Java 开发工程师），识别能力要求变化，明确标注 **新增 / 删除 / 修改** 的能力项，提供更新说明与数据源，支持人工优化与动态演化。

**实现流程**（`services/evolution.py`）：
1. 取岗位当前能力快照（旧版本）；
2. 解析输入的最新 JD，与历史能力合并进行交叉验证聚合；
3. **差异计算**：
   - **新增（add）**：新数据中出现、旧图谱中没有的能力项，附独立来源数与置信度；
   - **删除（delete）**：旧图谱有、近期 JD 不再要求的过时能力（标记 deprecated，保留历史）；
   - **修改（modify）**：重要度（必备↔加分）或权重显著变化（阈值 0.2）；
4. 每条变更记录 `reason`（更新说明）与 `data_source`（数据源），岗位版本号 +1，全部写入 `capability_change` 表，前端时间线展示。

**示例**：用含"大语言模型应用 / RAG / 向量数据库 / Kubernetes"的新 JD 驱动 Java 工程师演化，系统自动新增"大语言模型、检索增强生成、容器化部署"等能力项并给出溯源说明。

### 3.3 新一代信息技术岗位全景图谱

**目标**：展示领域内岗位的能力要求，颗粒度到 **"技能点"** 级别，可按 **技术栈** 与 **级别 / 置信度** 切换视图。

**实现**（`services/graph_service.py` + 前端 ECharts 力导向图）：
- **节点**：岗位节点（按技术栈着色，新岗位琥珀描边）+ 技能点节点（大小随关联度变化）；
- **关系边**：岗位-技能（必备/加分，按重要度与置信度着色加权）；技能-技能（先修 prerequisite / 相关 related / 驱动 drives）；
- **交互**：拖拽、缩放、邻接聚焦、点击下钻（岗位→画像，技能→关联岗位）；
- **视图切换**：技术栈筛选（人工智能/大数据/智能系统/物联网/云计算与工程）、级别筛选、置信度阈值滑块（过滤低置信能力项）。

当前图谱规模：**29 个岗位、105 个技能点、288 能力关系、52 条技能先修/相关/驱动关系**。

### 3.4 人岗匹配诊断与差距分析

**目标**：用户输入技能或上传简历（PDF/Word），系统对比目标岗位图谱，输出差距分析，提供针对性改进建议与学习路径规划。

**实现**（`services/resume.py` + `services/matching.py`）：
1. **简历解析**：pdfplumber（PDF）/ python-docx（Word）抽取文本 → 大模型结构化抽取技能要素 + 词典规则兜底，归一化同义词，**提取召回率 100%、F1 96.5%**；
2. **多维度匹配**：
   - 必备技能覆盖率（按权重）、加分技能覆盖率、技能级别匹配度、领域相关度；
   - **精确匹配（词典归一化）为主 + 语义匹配（BGE 向量）为辅**；
   - 综合匹配度 = 0.62×必备覆盖 + 0.16×加分覆盖 + 0.14×级别匹配 + 0.08×领域相关；
3. **差距分析**：列出已具备能力、缺失必备技能（能力缺口）、缺失加分技能；
4. **学习路径规划**：基于技能先修关系（prerequisite）拓扑排序 + 重要度优先，生成分步学习路径；
5. **改进建议**：大模型生成总体建议、各技能学习方向与资源、预计达标周期。

**匹配分类准确率 100%**（必备技能"具备/缺失"判定与真值完全一致）。

## 四、关键技术创新

### 4.1 多源异构数据清洗与交叉验证融合机制（解决时滞/噪音/抄袭/通胀）

赛题创新点①。`services/cleaning.py` + `services/ingest.py`：

| 问题 | 检测方法 | 处理 |
|------|----------|------|
| **抄袭/重复** | 精确 MD5 + **64 位 SimHash** 海明距离（阈值≤3）近似去重 | 标记 `is_duplicate`，**排除出交叉验证**，避免重复来源虚增可信度 |
| **时滞（lag）** | `publish_date` 与采集时间差 | 新鲜度指数衰减权重（半衰期 180 天），旧数据降权 |
| **噪音（noise）** | 技能同义词归一化字典 + 后缀裁剪（"微服务架构"→"微服务"） | 合并同义技能，降低噪音碎片 |
| **能力通胀（inflation）** | 单条 JD 技能数显著高于同岗位中位数（≥median×1.4 且超出≥4）**且** 非共识/冷门技能占比>35% | 标记 `inflation_flag`，通胀技能因缺乏交叉验证被自动过滤 |

**实测效果**：303 条 JD 中，植入的 **8 条抄袭样本全部检出（8/8）**、**13 条通胀样本全部检出（13/13）**。

### 4.2 能力"幻觉"防控机制（提升图谱构建科学性）

赛题创新点②，本系统最核心的创新。`services/hallucination.py`：

**核心思想**：单一来源（含大模型臆造）不可信；**只有被≥2 个独立来源交叉验证、或有外部网络证据支撑的能力项，才以高置信度进入图谱**，且每个能力项保留可溯源证据。

**置信度模型**（sigmoid 加权）：
```
confidence = σ( 2.4·支持率 + 0.45·min(独立来源数,5) + 0.6·平均新鲜度 + 1.1·外部证据 − 1.7 )
```

**三重防控**：
1. **交叉验证门槛**：必备技能要求 ≥2 个独立非重复来源，否则自动降级为加分技能（防止单源臆造的能力项被定义为"必备"）；
2. **置信度过滤**：低于阈值（0.45）的能力项标记为 `candidate`，不进入正式图谱（过滤潜在幻觉/噪音）；
3. **证据溯源**：每个能力项保留多条证据（原始 JD 片段 / 外部链接），前端可逐项展开追溯，实现 **可解释、可审计**。

**实测**：构建过程中过滤了数十项低置信"幻觉/噪音"能力项；典型如某 JD 单独出现的"量子计算"被自动从"必备"降级；图谱平均置信度 **92.7%**。

### 4.3 嵌入模型退化守卫（工程鲁棒性创新）

实践中发现 bge-small-zh 中文向量模型对 **纯英文技能名（Spark/Hive/Java/Go）返回退化向量**（不同词余弦≈1.0），会导致语义匹配把"相关但不同"误判为"等价"，掩盖真实能力缺口。

**解决方案**：语义匹配仅在"双方均含中文字符"时启用，并加 **退化守卫**（不同字符串余弦≥0.999 视为退化向量予以忽略）；英文同义词交由词典归一化处理。该守卫使 **人岗匹配分类准确率从 86.9% 提升到 100%**，体现了对大模型/向量模型局限性的工程化防控能力。

### 4.4 RAG 接地的新岗位发现

新岗位定义不依赖大模型"凭空想象"，而是 **先用 Tavily + Serper 两个独立来源检索真实证据，再让大模型基于证据生成**，并对每个技能做证据校验，从源头降低幻觉，使新兴岗位定义"有据可循"。

## 五、数据库设计（知识图谱 Schema）

| 表 | 作用 |
|----|------|
| `job` | 岗位节点：名称、技术栈、级别、is_new、核心职责、典型场景、置信度、版本 |
| `skill` | 技能点节点：归一化名、技术栈、类型（hard/tool/soft）、层级、向量 |
| `job_skill` | 岗位-技能关系：重要度、权重、掌握级别、**置信度、独立来源数、状态（active/deprecated）** |
| `skill_relation` | 技能-技能关系：prerequisite / related / drives |
| `raw_jd` | 原始多源 JD：来源、发布时间、**dedup_hash、simhash、is_duplicate、inflation_flag、lag_days、质量分** |
| `evidence` | 能力项证据（反幻觉溯源）：来源类型、片段、权重 |
| `capability_change` | 能力演化记录：变更类型、新旧值、更新说明、数据源、置信度 |
| `resume` / `match_result` | 简历解析结果 / 匹配诊断结果 |
| `tech_trend` | 技术趋势：热度、动量（技术 A 爆发驱动技能需求） |

## 八、总结与展望

本系统完整实现了赛题要求的全流程闭环，在 **多源交叉验证清洗** 与 **能力幻觉防控** 两大创新点上提出了可量化、可溯源的工程方案，三项核心指标均显著超过 90% 的达标线，并已稳定部署上线。

**可迁移性**：技术方案与岗位无关，可平滑迁移到医疗、金融、智能制造等更多新一代信息技术领域岗位；数据源可扩展更多招聘平台与权威信源；图谱可对接企业人才盘点与培训体系，形成"招聘—盘点—培养"闭环。

**未来方向**：引入时间序列的技能热度趋势预测、构建岗位间转岗路径推荐、接入更强的领域向量模型以增强语义匹配。
