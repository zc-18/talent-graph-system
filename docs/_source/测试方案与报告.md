## 二、测试数据集

### 2.1 岗位 JD 数据集（`docs/测试数据/seed_jds.json`）

- **规模**：379 条 JD，覆盖 **31 个**新一代信息技术岗位簇，横跨 **人工智能 / 大数据 / 智能系统 / 物联网 / 云计算与工程** 五大技术栈：
  Java 开发、机器学习、大数据开发、算法、自然语言处理、计算机视觉、大模型应用开发、推荐/搜索算法、知识图谱、自动驾驶系统、服务机器人、数字孪生、嵌入式软件、边缘计算、工业物联网、云原生、MLOps 等。
- **真实性设计**：每条 JD 在职责措辞、技能子集、动词表述、排版模板上充分变化（多模板 + 同义动词），贴近真实"相似但不相同"的招聘文本。
- **对抗样本（用于验证数据清洗）**：
  - **抄袭样本 8 条**：整段复制已有 JD 仅改公司名/来源（验证近似去重；系统另自动检出 3 条模板高度相似的近重复，共 11 条）；
  - **能力通胀样本 17 条**：在正常技能外注入 6 个与岗位明显无关的冷门技术词（区块链、量子计算、FPGA、同态加密等，验证通胀检测）。
- **真值标签**（`ground_truth.json`）：每条 JD 标注真实的必备/加分技能、是否抄袭、是否通胀，用于自动化评测。

### 2.2 简历测试集（`docs/测试数据/test_resumes.json`）

- **规模**：18 份简历，覆盖 10 类目标岗位，匹配度分布从高到低（含部分匹配、较大差距样本）。
- **鲁棒性设计**：刻意使用同义词/缩写（torch、k8s、NLP、ML、数仓…）考验"抽取 + 归一化"能力。
- **真值标签**：每份简历标注真实技能集合。

## 三、测试方法与用例

### 3.1 JD 解析准确率

- **方法**：对 371 条非重复 JD 调用大模型解析，抽取必备/加分技能并归一化，与真值标签比对。
- **指标**：精确率 P、召回率 R、F1、重要度分类正确率。
- **结果**（`eval_jd_result.json`）：

```
精确率 Precision : 0.986
召回率 Recall    : 0.980
F1               : 0.982
重要度分类正确率  : 0.978
```

### 3.2 简历提取准确率

- **方法**：对 18 份简历解析抽取技能，与真值比对（软技能不计精确率惩罚）。
- **结果**（`eval_resume_result.json`）：

```
精确率 Precision : 0.937
召回率 Recall    : 1.000
F1               : 0.965
```

### 3.3 人岗匹配准确率

- **方法**：对每份简历与其目标岗位图谱，逐"必备技能"判定"具备/缺失"，与真值（候选人真实技能 ∩ 岗位必备技能）比对。
- **结果**（`eval_match_result.json`）：

```
分类准确率 : 1.000
匹配 F1    : 1.000
```

### 3.4 数据清洗有效性

| 项目 | 植入 | 检出 | 准确 |
|------|------|------|------|
| 抄袭/重复 | 8 | 8（+3 自动检出近重复） | 100% |
| 能力通胀 | 17 | 17 | 100% |

### 3.5 单元测试与覆盖率

- **框架**：pytest + pytest-cov，**52 个用例全部通过**，覆盖清洗、归一化、反幻觉聚合、匹配、演化、JD/简历/新岗位定义抽取后处理、API 端点等。
- **覆盖率：66%（≥60% 达标）**。

```
TOTAL  1408 stmts  476 miss  66%
其中核心服务：cleaning 98% / hallucination 97% / taxonomy 98% / matching 87% / extraction 88% / discovery 71% / resume 57%
```

### 3.6 既有岗位能力动态演化验证（核心功能②）

- **方法**：以 **Java 开发工程师** 为例，输入 2026 年最新招聘 JD 窗口 + 多源联网证据，经交叉验证识别能力要求变化，自动标注「新增 / 删除 / 修改」并溯源（落库 `capability_change`，岗位版本 v1 → v2）。
- **在线可见**：`/evolution` 页面历史记录 + 岗位详情「演化历史」标签；数据见 `docs/测试数据/既有岗位_Java开发工程师_图谱与数据源.json` 的 `能力演化示例_v1到v2` 字段。
- **结果**（合计 **新增 4 / 修改 2 / 删除 1**，每条均带更新说明与数据源）：

| 变更 | 能力项 | 更新说明 | 数据源 |
|------|--------|----------|--------|
| 🟢 新增 | 大语言模型 | Java 后端开始要求对接大模型能力中台 / AI 网关 | 5 独立来源 + 联网佐证 |
| 🟢 新增 | 检索增强生成（RAG）| 知识库问答 / 智能客服场景驱动后端集成 | 4 独立来源 + 联网佐证 |
| 🟢 新增 | 向量数据库 | 伴随 RAG 落地成为后端新增数据组件 | 4 独立来源 + 联网佐证 |
| 🟢 新增 | 云原生 | 云原生架构（Service Mesh/Serverless）持续渗透后端 | 5 独立来源 + 联网佐证 |
| 🟡 修改 | Kubernetes | 容器化部署由「加分」升级为「必备」 | 9 独立来源 |
| 🟡 修改 | 微服务 | 需求热度上升（权重 0.74 → 0.95）| 9 独立来源 |
| 🔴 删除 | JSP | 前后端分离 + Spring Boot 普及后过时淘汰 | 最新 JD 窗口连续未出现 |

> 验证「既有岗位能力动态更新」闭环：变更项可量化、可溯源、可人工优化，岗位定义随市场数据持续进化。

## 四、复现方式

```bash
# 1) 生成数据集
uv run python data/generate_dataset.py
uv run python data/generate_resumes.py

# 2) 构建图谱（pipeline：清洗→解析→交叉验证→落库）
uv run python data/run_pipeline.py --reset
uv run python data/seed_relations.py
uv run python data/seed_evolution.py   # 既有岗位 v1→v2 能力演化示例（增/删/改 + 溯源）

# 3) 准确率评测（三项核心指标）
uv run python data/evaluate.py all

# 4) 单元测试 + 覆盖率
uv run pytest --cov=app --cov-report=term-missing
```
