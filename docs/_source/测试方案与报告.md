## 二、测试数据集

### 2.1 岗位 JD 数据集（`docs/测试数据/seed_jds.json`）

- **规模**：303 条 JD，覆盖 25 个新一代信息技术岗位簇（人工智能 / 大数据 / 物联网 / 云计算与工程）：
  Java 开发工程师、机器学习工程师、大数据开发工程师、算法工程师、自然语言处理工程师、计算机视觉工程师、物联网开发工程师、数据分析师、后端开发工程师、Python 开发工程师。
- **真实性设计**：每条 JD 在职责措辞、技能子集、动词表述、排版模板上充分变化（多模板 + 同义动词），贴近真实"相似但不相同"的招聘文本。
- **对抗样本（用于验证数据清洗）**：
  - **抄袭样本 8 条**：整段复制已有 JD 仅改公司名/来源（验证近似去重）；
  - **能力通胀样本 13 条**：在正常技能外注入 6 个无关/冷门技能（验证通胀检测）。
- **真值标签**（`ground_truth.json`）：每条 JD 标注真实的必备/加分技能、是否抄袭、是否通胀，用于自动化评测。

### 2.2 简历测试集（`docs/测试数据/test_resumes.json`）

- **规模**：18 份简历，覆盖 10 类目标岗位，匹配度分布从高到低（含部分匹配、较大差距样本）。
- **鲁棒性设计**：刻意使用同义词/缩写（torch、k8s、NLP、ML、数仓…）考验"抽取 + 归一化"能力。
- **真值标签**：每份简历标注真实技能集合。

## 三、测试方法与用例

### 3.1 JD 解析准确率

- **方法**：对 295 条非重复 JD 调用大模型解析，抽取必备/加分技能并归一化，与真值标签比对。
- **指标**：精确率 P、召回率 R、F1、重要度分类正确率。
- **结果**（`eval_jd_result.json`）：

```
精确率 Precision : 0.988
召回率 Recall    : 0.984
F1               : 0.986
重要度分类正确率  : 0.984
```

### 3.2 简历提取准确率

- **方法**：对 18 份简历解析抽取技能，与真值比对（软技能不计精确率惩罚）。
- **结果**（`eval_resume_result.json`）：

```
精确率 Precision : 0.937
召回率 Recall    : 1.000
F1               : 0.965
```

### 3.3 人岗匹配准确率

- **方法**：对每份简历与其目标岗位图谱，逐"必备技能"判定"具备/缺失"，与真值（候选人真实技能 ∩ 岗位必备技能）比对。
- **结果**（`eval_match_result.json`）：

```
分类准确率 : 1.000
匹配 F1    : 1.000
```

### 3.4 数据清洗有效性

| 项目 | 植入 | 检出 | 准确 |
|------|------|------|------|
| 抄袭/重复 | 8 | 8 | 100% |
| 能力通胀 | 13 | 13 | 100% |

### 3.5 单元测试与覆盖率

- **框架**：pytest + pytest-cov，**45 个用例全部通过**，覆盖清洗、归一化、反幻觉聚合、匹配、演化、抽取后处理、API 端点等。
- **覆盖率：64%（≥60% 达标）**。

```

其中核心服务：cleaning 98% / hallucination 97% / matching 87% / taxonomy 98% / extraction 88%
```
