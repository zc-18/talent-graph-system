# 演示视频脚本（10 分钟以内）

> 智岗图谱（TalentGraph AI）· 题目 XH-202621
> 在线系统：http://101.200.184.201:8200/　|　建议录屏 1920×1080，配音 + 字幕

总时长目标：**9 分 30 秒**。按"背景痛点 → 全景图谱 → 新岗位发现 → 既有岗位能力更新 → 人岗匹配诊断 → 创新点收束"组织，**重点突出新岗位与既有岗位能力更新的图谱演示**（赛题明确要求）。

---

### 镜头 0 ｜ 开场（0:00–0:40）

- **画面**：系统首页"数据驾驶舱"（`/`）。
- **旁白**："数字经济时代，技术迭代远超人才培养周期，企业'招不到合适的人'，青年'看不清职业路径'。我们打造了**智岗图谱**——一个数据驱动、可自我进化的'人才能力大脑'。"
- **操作**：鼠标缓慢划过 4 个 KPI 卡（岗位 35、技能点 101、已处理 JD 379、抄袭拦截 11），停在"图谱平均置信度 93.0%"仪表盘与底部"全流程闭环"六步图。

### 镜头 1 ｜ 全景能力图谱（0:40–2:10）

- **画面**：进入"全景能力图谱"（`/panorama`）。
- **旁白**："这是新一代信息技术岗位的全景能力图谱，颗粒度到**技能点**级别。彩色节点是岗位，灰色节点是技能点，连线代表能力要求与置信度。"
- **操作**：
  1. 拖拽缩放图谱，展示力导向布局；
  2. 用顶部下拉切换技术栈（人工智能 / 大数据 / 智能系统 / 物联网 / 云计算与工程）与岗位级别，展示"按技术栈与级别切换视图"；
  3. 拖动"置信度≥"滑块，展示低置信能力项被过滤（反幻觉）；
  4. 点击"机器学习工程师"节点 → 右侧详情卡 → 点击技能"PyTorch"节点 → 展示该技能关联的多个岗位（下钻）。

### 镜头 2 ｜ 新岗位发现与定义（2:10–4:00）⭐

- **画面**："新岗位发现"（`/discovery`）。
- **旁白**："面对市场上正在萌芽的新岗位，系统通过 **Tavily + Serper 两个独立来源**联网检索真实证据，再让大模型**基于证据**生成定义——而不是凭空想象，从源头防止幻觉。"
- **操作**：
  1. 点击种子标签"AI 智能体开发工程师"（或输入框输入），点击"发现并定义"；
  2. 等待结果，展示左侧生成的岗位定义：**岗位名称、核心职责、必备技能（带置信度、'已验证'标记）、加分技能、典型行业应用场景**；
  3. 右侧展示"多源证据"列表与"独立来源数"，点开一条证据链接；
  4. 点击"保存到图谱"，跳转到该新岗位详情页，强调全景图谱中以**琥珀色高亮**的新岗位节点。
- **旁白**："生成的每一项能力都可溯源、可人工优化、支持动态更新。"

### 镜头 3 ｜ 既有岗位能力动态更新（4:00–5:50）⭐

- **画面**："岗位能力演化"（`/evolution`）。
- **旁白**："对既有岗位——比如 Java 开发工程师——技术在变，能力要求也在变。系统用最新招聘 JD 驱动能力演化。"
- **操作**：
  1. 岗位下拉选择"Java 开发工程师"；
  2. 文本框已预置一条含"Spring Cloud 微服务、Kubernetes、**大语言模型应用、RAG 检索增强、向量数据库**"的最新 JD；
  3. 点击"识别能力变化并演化"；
  4. 右侧展示统计卡：**新增 X / 删除 Y / 修改 Z**；逐条展示变更：
     - 🟢 新增"大语言模型 / 检索增强生成 / 容器化部署"——附**更新说明与数据源**；
     - 🟡 修改某技能重要度/权重——展示"必备↔加分"或权重变化箭头；
  5. 进入该岗位详情页 → "演化历史"标签 → 展示时间线，强调"明确标注新增/删除/修改 + 溯源"。

### 镜头 4 ｜ 人岗匹配诊断与差距分析（5:50–8:00）

- **画面**："人岗匹配诊断"（`/match`）。
- **旁白**："求职者或 HR 上传简历，系统解析技能，对比目标岗位图谱，给出精准的差距诊断。简历仅在内存中解析用于本次匹配，原始简历与姓名等个人信息不在服务器留存（隐私合规）。"
- **操作**：
  1. 目标岗位选"机器学习工程师"；
  2. 上传一份 PDF/Word 简历（或切到"粘贴文本"演示），展示**解析出的技能要素**（强调 torch→PyTorch 等同义归一化）；
  3. 点击"开始匹配诊断"，展示结果：
     - **综合匹配度仪表盘** + 等级；
     - **多维度雷达图**（必备覆盖 / 加分覆盖 / 级别匹配 / 领域相关）；
     - **已具备 vs 能力缺口**（清晰标注缺失项）；
     - **岗位学习路径规划**（按先修关系拓扑排序的分步路径）；
     - **大模型针对性改进建议**（含资源方向与达标周期）。
- **旁白**："从'你差什么'到'怎么补'，给出可执行的成长路径。"

### 镜头 5 ｜ 创新点与溯源（8:00–9:10）

- **画面**：回到某岗位详情页 → "溯源证据"标签。
- **旁白**："系统的科学性来自两大创新：一是**多源交叉验证清洗**，自动识别招聘数据的抄袭、通胀、时滞与噪音；二是**能力幻觉防控**——只有被多个独立来源验证的能力才进入图谱，每一项都保留可追溯的证据。"
- **操作**：展开某能力项证据（独立来源数、原始 JD 片段）；切到"岗位库管理"快速展示 35 个岗位、各自置信度。
